Quantum Git

Master the essentials of version control with our futuristic guide.

1. git init

Initializes a new Git repository, transforming your current directory into a tracked project. This command creates a hidden .git folder, the core of your repository's history and configuration.

git init
Initialize Repository

2. git clone

Clones an existing remote repository onto your local machine. This is your entry point to a project, downloading all files, branches, and commit history from the specified URL.

git clone https://github.com/username/repository.git
Clone Remote Repo

3. git status

Displays the state of your working directory and staging area. Use this to see which files are modified, staged, or untracked, providing a clear overview before you commit.

git status
Check Status

4. git add

Moves changes from your working directory to the staging area. Think of this as preparing your changes for the next commit. You can add specific files or all changes at once.

git add filename.txt
git add . # Add all changes
Stage Files

5. git commit

Saves the staged changes permanently to your repository history. Every commit should have a descriptive message, acting as a snapshot of your project at that moment.

git commit -m "Descriptive commit message"
Create a Commit

6. git push

Transfers your local commits to a remote repository, such as GitHub. This command synchronizes your work with the shared project, making your changes visible to others.

git push origin main
Push to Remote

7. git pull

Fetches and merges changes from the remote repository into your current local branch. This is how you update your project with the latest work from your collaborators.

git pull origin main
Pull from Remote

8. git branch

Manages your project's branches. You can use it to list all existing branches or to create a new one, isolating new features or bug fixes from your main codebase.

git branch # List all branches
git branch new-feature # Create a new branch
Manage Branches

9. git checkout

Switches between branches or restores files. This is your primary tool for navigating different versions of your project and working on isolated features.

git checkout new-feature
Switch Branch

10. git merge

Combines the history of one branch into another. Once a feature is complete, you use this command to integrate its changes back into your main branch.

git merge new-feature
Merge Branches
